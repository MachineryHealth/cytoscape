BAM Event Model Release Notes (v0.2)

This version has an API for handling batching for efficiency.
  1. Various EventManager changes:
    1. removed the 3 argument dispatch()
    2. renamed dispatchDelayed() to dispatchAccumulated().
    3. added registerInterest (Class,...) and unregisterInterest (Class,...).
    4. added startAccumulating ().

  2. Various EventConditions changes:
    1. Removed isClassEvent().
    2. Removed getSynchModel().
    3. Added getDeliveryOptions().

  3. Added DeliveryTiming, DeliveryOptions, and DeliveryMethod.

  4. Changed EventReceiver-->Listener.

  5. Removed EventFactory.SINGLETON in favor of adding ConcreteEventFactory.getInstance().

    Placing a constant in EventFactory interface was poluting the interface.

  6. Added several JUnit tests.

    InstanceAndArgFeatureTest shows the existing core3 model along side BAM for creating an instance notification with multiple arguments.

    BatchTest shows the API with nested batches along with events received immediately (but does not solve the deletion problem).
